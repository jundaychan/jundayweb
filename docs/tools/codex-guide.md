# Codex 完全上手指南：让 AI 替你写代码

> OpenAI Codex 是一个**会自己动手写代码的 AI Agent**。你不用懂编程，把想做的东西用大白话讲清楚，它就能帮你写出来、跑起来。这篇是给"非程序员/编程小白"的中文上手指南。
>
> 想看作者原版（含每一步真实截图）：[gpt-codex 全网最详细教程（MaynorAI）](https://github.com/xianyu110/gpt-codex)

## 一、Codex 是什么，能帮老板干什么

把 Codex 想象成一个**住在你电脑里的初级程序员**。它和普通 ChatGPT 最大的区别是：ChatGPT 只会告诉你"代码应该这么写"，而 Codex 会**真的把代码写进文件、运行起来、自己改 bug**。

对不懂技术的老板，它能做这些事：

- 把"我想要一个自动统计每天订单的小工具"从一句话变成能跑的程序
- 做一个简单的网站、落地页、小程序原型
- 写自动化脚本：批量改文件名、合并表格、定时抓数据
- 帮你看懂、改动别人留下的老代码

> **一个反常识的观点**：编程小白往往比老程序员更适合用 Codex。因为你没有"必须自己写"的执念，更愿意把活完全交出去——而这恰恰是用好 AI Agent 的关键心态。

## 三、核心概念：项目 + 线程（最重要的一节）

Codex 用"两层结构"来组织你的工作，搞懂这个，你就不会越用越乱：

![Codex 的两层结构：项目与线程](/images/guides/codex-threads.svg)

- **项目文件夹（Project）**：一个具体的活，比如"公司官网""订单小工具"。它对应你电脑里的一个文件夹。
- **线程（Thread）**：一段独立的对话，专门处理这个项目里的**一件小事**，比如"改首页 banner"。

**黄金法则**：**一个线程只干一件事，干完就开新线程。** 别把"改首页 + 加表单 + 修 bug"全堆在一个对话里——堆久了 AI 会"记混"，越聊越糊涂。这和我们反复强调的"上下文要干净"是一个道理。

## 四、几个让 Codex 更好用的功能

- **定时任务**：让它每天/每周自动跑某个活，比如每天早上抓一次数据生成报表。
- **Skills（技能）**：把你常用的一套做法打包成"技能"，下次一句话就能调用，不用每次重新交代。还有个 `Skill Creator` 能帮你生成技能。
- **MCP / 记忆 / 云端运行 / 电脑自动化**：进阶能力，能让它连外部工具、记住你的偏好、在云端跑、甚至自动操作电脑。新手可以先放一放，用熟了再碰。

## 五、最关键的用法：Plan Mode（先想清楚再动手）

这是新手最该养成的习惯。直接让 AI"边想边干"，它容易一头扎进错误方向。正确做法是**先让它出方案、你看过点头，它再动手**。

```
你提需求  →  让它先出"计划"  →  你审一眼方案  →  确认无误  →  才让它执行
```

- 觉得它想得不对，就在计划阶段打回重来——这时候改方向几乎零成本。
- 任务越复杂，越要先 Plan。简单小事可以直接干。

还有个**推理深度**的设置：简单任务用浅一点（快、省），复杂任务调深一点（慢、准）。

## 六、动手实战：Vibe Coding 的几个习惯

"Vibe Coding"就是**你描述感觉和需求，AI 负责实现**的开发方式。几个实用习惯：

1. **需求讲具体**：别说"做个好看的页面"，说"做一个白底、顶部有 logo、中间一句话标语、底部留微信二维码的落地页"。
2. **小步快跑**：一次只让它改一小块，跑通了看一眼效果，再继续。别一次提十个要求。
3. **前端不满意就让它截图对比**：页面样子不对，把你想要的样子描述清楚，让它一版版调。
4. **管好代码、别弄乱项目**：让 Codex 改代码，但养成"改完就保存一个版本"的习惯（用 Git），万一改坏了能退回去。
