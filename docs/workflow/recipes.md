# 常用工作流配方

> 6 个拿来就能用的工作流模板。选一个最痛的场景，今天就搭起来。

## 配方一：每日数据报告

**适合**：电商老板、运营负责人

```
定时触发（每天 8:00）
  ↓
查询数据（数据库 / API / 表格）
  ↓
AI 分析（同比环比、异常检测、趋势判断）
  ↓
生成报告（文字 + 关键数据）
  ↓
推送通知（飞书群 / 钉钉群 / 邮件）
```

### AI 分析提示词模板

```
你是一位数据分析师。请分析以下销售数据，用 200 字以内总结：

1. 核心指标变化（对比昨天/上周同天）
2. 有无异常值（突然暴涨或暴跌的指标）
3. 一句话行动建议

数据：
{{data}}
```

### 省时效果

| 指标 | 手动 | 自动化后 |
|------|------|---------|
| 每日耗时 | 30-60 分钟 | 0 分钟 |
| 每月节省 | - | 15-30 小时 |
| 报告质量 | 取决于谁做 | 一致稳定 |

## 配方二：客户跟进提醒

**适合**：销售团队、客户成功团队

```
定时触发（每天 9:00）
  ↓
检查 CRM（哪些客户超过 N 天没跟进？）
  ↓
AI 生成跟进建议（基于客户历史）
  ↓
发送提醒给负责人
  ↓
可选：自动创建待办任务
```

### AI 跟进建议提示词

```
这位客户的信息：
- 名称：{{name}}
- 上次跟进：{{last_contact_date}}（{{days_ago}} 天前）
- 上次沟通内容：{{last_note}}
- 客户阶段：{{stage}}

请给出简短的跟进建议（50 字以内），包括：
1. 建议的切入话题
2. 需要注意的点
```

## 配方三：社交媒体内容生产

**适合**：电商运营、自媒体团队

```
触发方式：手动 / 定时 / 新品上架时
  ↓
获取素材（产品信息 / 热点话题）
  ↓
AI 生成内容（小红书文案 / 抖音脚本 / 朋友圈文案）
  ↓
人工审核（AI 生成的内容必须人工过一遍）
  ↓
定时发布 / 存入待发布队列
```

### 小红书文案提示词

```
你是一位小红书种草达人。请为以下产品写一篇小红书笔记：

产品：{{product_name}}
卖点：{{selling_points}}
目标人群：{{target_audience}}
价格：{{price}}

要求：
1. 标题含 emoji，能引起好奇
2. 正文口语化，像朋友推荐
3. 包含使用场景
4. 300 字左右
5. 最后加 5-8 个相关话题标签
```

::: warning 必须人工审核
AI 生成的社交媒体内容必须人工审核后再发布。AI 可能：
- 编造使用效果
- 违反广告法（"最"、"第一"等绝对化用语）
- 语气不符合品牌调性
:::

## 配方四：竞品监控

**适合**：运营、市场、产品团队

```
定时触发（每天 / 每周）
  ↓
抓取竞品信息（官网价格、新品上架、社交媒体动态）
  ↓
AI 分析变化（和上次对比有什么变化？）
  ↓
生成竞品动态报告
  ↓
推送给相关人员
```

### n8n 实现要点

1. **HTTP Request 节点**：抓取竞品网页
2. **HTML Extract 节点**：提取关键信息（价格、标题）
3. **Compare 节点**：和上次抓取的数据对比
4. **AI 节点**：分析变化并生成洞察
5. **通知节点**：有变化才通知（避免每天发废话）

## 配方五：邮件/工单自动处理

**适合**：客服团队、运营团队

```
新邮件 / 新工单进来
  ↓
AI 分类（咨询 / 投诉 / 退货 / 合作 / 垃圾）
  ↓
根据类型自动处理：
  ├── 咨询 → 检索知识库 → AI 草拟回复 → 发给客服确认
  ├── 投诉 → 标记紧急 → 通知负责人
  ├── 退货 → 创建退货工单 → 发退货指引
  ├── 合作 → 转发给商务
  └── 垃圾 → 归档
```

### 分类提示词

```
请判断以下邮件/消息属于哪个类别，只返回类别名称：
- 咨询（产品信息、使用方法、价格等）
- 投诉（不满意、质量问题、服务差等）
- 退货（退货、退款、换货等）
- 合作（商务合作、代理、批发等）
- 垃圾（广告、推销、无关内容等）

邮件内容：
{{email_content}}
```

## 配方六：周报/月报自动生成

**适合**：团队管理者

```
定时触发（每周五 17:00 / 每月最后一天）
  ↓
收集数据（多来源汇总）：
  ├── 项目管理工具（完成的任务数）
  ├── CRM（新增客户、成交额）
  ├── 客服系统（工单量、满意度）
  └── 财务系统（收入、支出）
  ↓
AI 汇总分析（本周亮点、问题、下周重点）
  ↓
生成结构化报告
  ↓
发送（邮件 / 飞书文档 / 钉钉）
```

### 周报提示词

```
请基于以下数据生成本周工作周报，300 字以内：

## 数据
{{data}}

## 格式要求
1. 本周亮点（2-3 条）
2. 存在问题（1-2 条）
3. 下周重点（2-3 条）

语气：简洁、专业、有数据支撑。
```

## 搭建工作流的 5 个原则

| 原则 | 说明 |
|------|------|
| **从最痛的开始** | 先自动化最耗时、最重复的任务 |
| **先简后繁** | 先做 3 步的简单流程，跑通了再加步骤 |
| **关键步骤留人工** | 发布内容、退款审批等关键动作，保留人工确认 |
| **加监控** | 工作流失败时要有通知，不能默默出错 |
| **定期维护** | API 会变、数据格式会变，每月检查一次 |

## 工作流 → Agent 的升级路径

当你的工作流越来越复杂（超过 10 个节点、有很多条件分支），可以考虑升级为 Agent：

```
简单工作流（3-5 步）
  → 你现在在这里，固定流程就够了

复杂工作流（10+ 步，多分支）
  → 考虑把部分步骤交给 Agent 自主判断

多个工作流需要协调
  → 考虑用 Agent Harness 来管理
```

**波兰尼（Polanyi）的洞察**：工作流设计的难点不在工具操作，而在于"预判所有可能的异常"——这本身就是一种隐性知识。你在实际跑工作流的过程中，才会逐渐发现哪些边界情况需要处理。所以先跑起来，再迭代。

## 一句话总结

> **选一个最痛的重复性任务，今天就用工作流自动化。省下来的时间，拿去做真正需要人脑子的事。**
