# 部署与运维

> 代码写完还要上线运行，这些概念帮你理解从开发到上线的最后一公里。

## 75. Docker（容器化）

将应用及其运行环境打包成标准化"容器"的技术，确保应用在任何服务器上都能一致运行。

## 76. 云服务（Cloud Services）

通过互联网租用计算资源和服务，如 AWS、Azure、阿里云，无需自己购买和维护服务器。

## 77. Serverless（无服务器架构）

不需要管理服务器，代码按需运行、按使用量付费的云计算模式，适合 AI 应用的弹性需求。

## 78. Vercel / Netlify

专注于前端和全栈应用部署的云平台，支持一键部署，与 GitHub 集成实现自动化发布。

## 79. 环境变量（Environment Variables）

存储 API 密钥、数据库密码等敏感配置信息的方式，避免将密钥直接写在代码中泄露。

## 80. API 密钥管理（API Key Management）

安全地生成、存储和轮换 AI 服务访问密钥的实践，防止密钥泄露导致的经济损失。

## 81. 成本监控（Cost Monitoring）

跟踪和管理 AI API 调用量和费用的过程，避免因调用量激增而产生意外的高额账单。

## 82. 速率限制（Rate Limiting）

限制一定时间内 API 调用次数的机制，防止滥用并控制成本。

## 83. 日志与监控（Logging & Monitoring）

记录应用运行状态和 AI 模型输出的系统，用于排查问题和优化性能。

## 84. 域名与 HTTPS（Domain & HTTPS）

为你的 AI 应用配置专属域名和安全证书，让用户通过安全的网址访问你的服务。
