AI 学习路线

普通人的AI进化路径
跟你想的不一样

为什么写Skill比设计工作流简单？为什么接MCP比拖低代码容易？

大多数人以为的路径

对话 → 工作流编排 → Agent → 系统

实际走下来的路径

1

学会对话

理解AI是什么，学会给它下指令

2

用现成能力

选对工具，处理实际业务

3

写自己的 Skill 比想象中简单

把你脑子里"知道怎么做但从没写下来"的经验写成文档

4

接 MCP 边界清晰

告诉AI可以用什么工具，输入什么输出什么

5

搭 Agent

让AI自主决策和执行任务

6

系统构建

用AI重构业务流程和组织

赋能（声明式）

告诉AI"我会什么"

  • 写Skill = 写文档
  • 接MCP = 定义接口
  • 确定性高，边界清晰
  • 把默会知识变成AI能力

编排（命令式）

替AI设计"该怎么做"

  • 要处理所有失败路径
  • 并发、状态、异常
  • 得撞过才知道哪里有坑
  • 正在被Agent吃掉

核心洞察

声明"我会什么"
比设计"该怎么做"简单。
赋能比编排更有杠杆。

在AI native时代，你不该操心流程，你该操心给AI配什么能力。工作流编排正在被Agent吃掉。

我们知道的永远比我们能说出来的多。写Skill就是在做这件事——把默会知识变成AI能用的显性知识。而工作流编排之所以难，恰恰因为"预判所有意外"的能力，本身就是最难言传的默会知识。

— Michael Polanyi 默会知识理论
