## n8n：最灵活的开源方案

### 为什么选 n8n

- **700+ 预建集成**：邮件、数据库、微信、飞书、钉钉、Notion、Google Sheets...
- **可视化拖拽**：连线式设计，一目了然
- **开源可自部署**：数据完全掌控
- **中文社区**：[n8n.akashio.com](https://n8n.akashio.com) 有中文文档和教程

### 部署 n8n

```bash
# Docker 一键部署
docker run -d \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n
```

访问 `http://localhost:5678`，注册管理员账号。

### 实战：每日销售数据报告

**目标**：每天早上 8 点自动从数据库拉昨日数据，AI 写分析报告，发到飞书群。

#### 第一步：创建工作流

点击"New Workflow"，然后逐个添加节点：

#### 第二步：添加定时触发

- 搜索添加 **Schedule Trigger** 节点
- 设置：每天 08:00 触发

#### 第三步：查询数据库

- 添加 **MySQL**（或 PostgreSQL）节点
- 配置数据库连接
- 写查询语句：

```sql
SELECT 
  DATE(created_at) as date,
  COUNT(*) as order_count,
  SUM(amount) as total_amount,
  AVG(amount) as avg_amount
FROM orders
WHERE created_at >= CURDATE() - INTERVAL 1 DAY
  AND created_at < CURDATE()
```

#### 第四步：AI 分析

- 添加 **OpenAI**（或通义千问）节点
- 提示词：

```
分析以下销售数据，200 字以内：
- 和前一天对比（上升/下降/持平）
- 有没有异常值
- 一句话总结今日重点

昨日数据：
订单数：{{$json.order_count}}
总销售额：{{$json.total_amount}}
平均客单价：{{$json.avg_amount}}
```

#### 第五步：发送到飞书

- 添加 **HTTP Request** 节点
- URL：你的飞书群 Webhook 地址
- Method：POST
- Body：

```json
{
  "msg_type": "text",
  "content": {
    "text": "📊 昨日销售报告\n\n{{$json.output}}"
  }
}
```

#### 第六步：测试

点击"Execute Workflow"手动触发一次，检查每个节点的输出。

## Dify 工作流：RAG + 工作流一站式

### 为什么选 Dify 工作流

- 如果你已经用 Dify 做了 RAG 知识库，工作流直接复用
- 可以在工作流中调用知识库检索
- 适合"先检索知识，再做决策"的场景

### 实战：智能客服分流

**目标**：客户消息进来，AI 自动判断类型，简单问题自动回答，复杂问题转人工。

#### 创建工作流应用

1. Dify → "工作室" → "创建应用" → 选择"工作流"
2. 工作流画布会打开

#### 设计流程

```
客户消息进来
  ↓
[LLM 节点] 判断问题类型（FAQ/投诉/咨询/其他）
  ↓
[条件分支] 
  ├── FAQ → [知识库检索] → [LLM 节点] 基于检索结果回答 → 输出
  ├── 投诉 → 输出"正在为您转接专属客服，请稍候"
  └── 其他 → [LLM 节点] 通用回答 → 输出
```

#### 关键节点配置

**分类节点**（LLM）：

```
请判断以下客户消息属于哪个类型，只返回类型名称：
- FAQ（产品信息、退换货、物流等常见问题）
- 投诉（不满意、要求赔偿、态度差等）
- 咨询（购买建议、定制需求等）
- 其他

客户消息：{{input}}
```

**知识库检索节点**：绑定你的 FAQ 知识库，Top K 设为 3。

**回答节点**（LLM）：

```
基于以下参考信息回答客户问题，简洁友好，不超过 100 字：
{{context}}
客户问题：{{input}}
```

## 飞书多维表格：零门槛自动化

### 为什么选飞书

- **零部署**：飞书企业版自带
- **零代码**：界面操作，不用写一行代码
- **AI 公式**：2026 年新功能，在表格里直接调 AI

### 实战：客户跟进自动提醒

**目标**：CRM 表格中的客户，如果 3 天没跟进，自动发提醒给负责人。

#### 第一步：建多维表格

字段设计：

| 字段 | 类型 | 说明 |
|------|------|------|
| 客户名称 | 文本 | |
| 负责人 | 人员 | 飞书成员 |
| 最后跟进日期 | 日期 | |
| 状态 | 单选 | 跟进中/已成交/已流失 |
| 需要提醒 | 公式 | =IF(AND(状态="跟进中", 今天-最后跟进日期>3), "是", "否") |

#### 第二步：创建自动化

1. 点击表格右上角"自动化" → "新建自动化"
2. 触发条件：每天 09:00
3. 筛选条件：`需要提醒 = "是"`
4. 动作：发送飞书消息给对应的"负责人"

消息模板：

```
⏰ 客户跟进提醒

客户 {{客户名称}} 已经 {{今天-最后跟进日期}} 天没有跟进了。
请今天安排联系。
```

#### 第三步：AI 增强（可选）

飞书多维表格支持 AI 字段，可以添加：
- **AI 摘要字段**：自动总结历史跟进记录
- **AI 建议字段**：基于客户情况给出跟进建议

## 钉钉宜搭：钉钉生态首选

### 实战：审批后自动通知 + AI 摘要

**场景**：合同审批通过后，自动发钉钉消息给相关人员，并用 AI 生成合同要点摘要。

1. 在宜搭中创建审批流程
2. 添加"通过后"触发器
3. 添加"AI 摘要"节点（调用通义千问）
4. 添加"钉钉消息"节点，发送摘要给相关人

> IDC 2025 年报告将钉钉宜搭列为中国低代码平台市场的领导者之一。
