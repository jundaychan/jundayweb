## 一、Codex 是什么，能帮老板干什么

把 Codex 想象成一个**住在你电脑里的初级程序员**。普通 ChatGPT 只会告诉你"代码该这么写"，而 Codex 会**真的把代码写进文件、运行起来、自己改错**。

对不懂技术的老板，它能做这些事：

- 把"我想要一个自动统计每天订单的小工具"从一句话变成能用的程序
- 做一个简单的网站、落地页、活动页
- 写自动化小工具：批量改文件名、合并表格、定时抓数据
- 帮你看懂、改动别人留下的老代码

> **一个反常识的观点**：编程小白往往比老程序员更适合用 Codex。因为你没有"必须自己写"的执念，更愿意把活完全交出去——而这恰恰是用好 AI 的关键心态。

## 二、怎么获取（国内三种方式，含价钱）

![获取 Codex 的三种方式](/images/guides/codex-access.svg)

| 方式 | 大概要花多少 / 怎么付 | 适合谁 |
|------|------|--------|
| **① 官网订阅** | 约 20 美元/月起，**需要海外信用卡**，且要科学上网 | 有海外卡 + 网络条件的人 |
| **② 国内中转**（新手推荐） | 注册一个国内中转服务，**通常可微信/支付宝充值**，按用量付费 | 怕折腾、没海外卡的老板 |
| **③ cc-switch 工具** | 一个帮你在不同渠道间切换的小工具 | 进阶用户，新手先忽略 |

::: warning 关于"科学上网 / 翻墙"
Codex 官方在中国大陆连不上，官网方式需要科学上网。用 VPN 翻墙访问国外平台**违反其服务条款、也有合规风险**。所以新手更推荐走**国内中转渠道**——不用翻墙、能用人民币付，省心。具体选哪家中转、怎么避免被坑，建议优先选有口碑、能开发票的服务，别图便宜用来路不明的小站。
:::

## 三、第一次使用：从下载到打开

1. **下载安装**：Codex 有电脑客户端（苹果 macOS 和 Windows 普通电脑都有）。下载好安装包后，**像装微信一样，双击、一路点"下一步/同意"即可**，中途不会让你做技术选择。
2. **打开登录**：装好后双击打开，用你的账号登录（官方账号需先在 OpenAI 注册；走中转的话按中转服务的说明登录）。
3. **它长什么样**：打开后是一个**带聊天框的窗口**——你在框里用中文打字给它下任务，它一边回你、一边动手。

> 装好打不开、连不上，多半是网络或账号问题。新手卡在这一步很正常，先确认走的是哪种渠道（官方 or 中转），按对应说明排查。

## 四、核心概念：一个项目，多个"聊天窗口"

Codex 用"两层结构"组织工作，搞懂这个就不会越用越乱：

![Codex 的两层结构：项目与线程](/images/guides/codex-threads.svg)

- **项目文件夹**：一个具体的活，比如"公司官网""订单小工具"。它对应你电脑里的一个文件夹。
- **线程（其实就是"一个聊天窗口"）**：一段独立对话，专管这个项目里的**一件小事**，比如"改首页图片"。

**黄金法则**：**一个聊天窗口只干一件事，干完就开新的。** 别把"改首页 + 加表单 + 修毛病"全堆在一个窗口里——堆久了它会"记混"，越聊越糊涂。

## 五、最关键的用法：先让它列计划，你点头它再动手

这是新手最该养成的习惯。直接让它"边想边干"容易跑偏。正确做法是**先出方案、你看过点头，它再动手**（这个就叫 Plan 模式 / 计划模式）。

```
你提需求  →  让它先列"计划"  →  你瞄一眼方案  →  确认没问题  →  才让它动手
```

- 觉得它想得不对，就在计划阶段打回重来——这时候改方向几乎不费成本。
- 越复杂的活越要先列计划；简单小事可以直接干。

## 六、几个让 Codex 更省事的功能

![Codex 定时任务：按点自动干活](/images/guides/codex-schedule.svg)

- **定时任务**：让它每天/每周自动跑某个活，比如每天早上抓一次数据、生成报表。

![Skills：把常用做法打包，一句话复用](/images/guides/codex-skills.svg)

- **Skill（技能）**：把你常用的一套做法打包，下次一句话就能调用，不用每次重新交代。
- **MCP / 记忆 / 云端运行等**：进阶功能，**新手完全不用管，跳过即可**，用熟了再回来看。

## 七、动手实战：你说需求、它来实现

几个让结果更好的习惯：

1. **需求讲具体**：别说"做个好看的页面"，说"做一个白底、顶部放 logo、中间一句标语、底部留微信二维码的页面"。
2. **小步快跑**：一次只让它改一小块，跑通看一眼，再继续。别一次提十个要求。
3. **不满意就让它一版版调**：把你想要的样子描述清楚，让它改到你满意。
4. **记得"存档"**：让它改东西前，养成保存一个版本的习惯，万一改坏了能退回去（进阶可学 Git，**新手可先用它界面里的"历史版本/撤销"功能，不用学 Git**）。

## 八、安全 & 搞砸了怎么办（老板必看）

- **它会读写你授权的文件夹**——所以**单独建一个文件夹给它用**，里面只放这个项目相关的东西，别放客户名单、账本、合同等敏感资料。
- **它默认会先问你再执行**关键操作，你看到不对就别点同意。
- **改坏了能退回**：配合上面说的"存档/历史版本"，可以回到上一个好的版本，它不会动你其他文件。
- **花钱可控**：按用量付费的渠道，建议先小额充值试水，跑顺了再加。
