# Claude Code 完全上手指南：技术圈公认最强的编程 Agent

> Claude Code 是 Anthropic 出品、在**终端里运行**的编程 Agent，由 Claude 模型驱动。它能读懂你**整个项目**、跨多个文件干活，被很多程序员认为是目前编程能力最强的 AI 工具。这篇讲它是什么、国内怎么用、怎么用得好。
>
> 参考：[Claude Code 官方文档](https://code.claude.com/docs/en/overview) · [国内使用中文指南](https://github.com/claude-code-chinese/claude-code-guide)

## 一、它和普通 AI、和 Codex 有什么不一样

- **不是聊天框，是干活的**：它直接读你的代码、改文件、运行命令、处理 Git，全程用大白话指挥。
- **理解整个项目**：不是只看你贴的一段代码，而是能把整个代码库当成上下文，跨文件改动。
- **能力强、但偏技术**：相比对小白更友好的 [Codex](/tools/codex-guide)，Claude Code 更"硬核"，但上限更高。Anthropic 自己内部团队大量用它开发产品。

## 二、国内怎么用（最关键的一步）

Claude Code 在中国大陆无法直接访问。国内最常见的上手方式是**用 API 代理站**：注册一个国内中转服务，拿到 API key 和接入地址，填进配置里即可。对新手，这是绕过限制最省事的路子。

**大致安装流程**（Mac / Windows / Linux 都支持）：

1. 装好运行环境（Node.js 等基础组件，一键脚本能帮你搞定）
2. 通过 npm 安装 Claude Code 命令行工具
3. 在终端输入版本号命令，能正确显示版本 = 安装成功
4. 把你的 API key 和接入地址填进配置文件（全局配置或单个项目配置都行）

> 中文指南里附了**一键安装脚本**，把 API key 和地址填进去就能自动装好，新手强烈建议用它，能省掉一堆环境折腾。详见 → [claude-code-guide](https://github.com/claude-code-chinese/claude-code-guide)。

::: tip 配置小贴士
- **权限设置**：可以配置哪些操作需要你点头确认，哪些可以放手自动做。新手建议一开始让它"多问几句"，熟了再放权。
- **工作目录**：指定它在哪个文件夹里干活，别让它满电脑乱跑。
:::

## 三、用好它的三条核心心法

![Claude Code 的正确姿势：先规划再动手](/images/guides/claude-code-flow.svg)

这三条来自 Anthropic 官方最佳实践，是新手和老手差距最大的地方：

### 1. 先规划，再动手（最重要）

别一上来就让它写。先用 **Plan 模式**让它把"打算怎么做"讲清楚，你看过、确认方向对了，再放它去执行。任务越复杂，这一步越值钱——它能避免 AI 朝错误方向猛干半天。

### 2. 管好"上下文"，别让它记太多反而糊涂

AI 的"注意力"是有限的，对话堆太长反而会变笨：

- 一件事干完了，用 `/clear` **清空重置**，开始下一件事
- 对话太长时，用 `/compact` 让它**把关键决定压缩成摘要**，留住重点、丢掉啰嗦
- 提问时用 `@` **直接引用某个文件**，比你复制粘贴一大段更准

### 3. 给项目放一份"说明书"：CLAUDE.md

在项目根目录放一个叫 `CLAUDE.md` 的文件，写清楚：这个项目是干嘛的、有什么规矩、用什么技术、不要碰哪些东西。Claude Code **每次都会先读它**，相当于给新员工一份入职手册——交代一次，往后都省心。

## 四、典型用法举例

- "帮我在这个网站加一个'联系我们'的表单" → 它读懂现有结构，自己找到该改的文件
- "这个页面在手机上显示错位了，修一下" → 它定位问题、改样式、让你看效果
- "把这个老项目的依赖升级到最新版，别改坏功能" → 跨文件批量处理，还能帮你跑测试验证

## 五、Codex 还是 Claude Code？

| | Claude Code | Codex |
|---|---|---|
| 能力上限 | 公认更强 | 强，对小白更友好 |
| 上手门槛 | 偏技术 | 有专门的小白中文教程 |
| 适合 | 想要最强效果 | 第一次接触编程 Agent |

不冲突，预算够两个都装来对比。Codex 的玩法看 → [Codex 完全上手指南](/tools/codex-guide)。

如果你或团队完全不写代码、只想让 AI 帮你处理办公文档，那这两个都不是首选，去看国内能直接用的 → [腾讯 WorkBuddy 指南](/tools/workbuddy-guide)。
